The core resolves a MotionDoc to a tree of final numbers. An **adapter** is a thin painter
that walks that tree and draws it for one backend. Because they all paint the *same* tree, an
animation looks and **times identically** wherever it runs.

## Paint backends

These packages actually draw pixels / DOM / native views:

## Framework bindings

These wrap **dom** and/or **canvas** with idiomatic components / actions / elements.
Playback surface matches React: play, pause, seek, `progress` (0…1), rate.

## Meta-frameworks (no extra package)

Use an existing adapter inside the host framework:

| Stack | Use |
| --- | --- |
| **Next.js** | `@blinn-motion/react` in a **client** component (`"use client"`) |
| **Astro** | React / Lit **islands** (`client:load`) — no server paint required |
| **Expo** | `@blinn-motion/react-native` |
| **Vite vanilla** | `@blinn-motion/dom` or `@blinn-motion/canvas` |

Runnable demos live under [`examples/`](https://github.com/ismailyagci/blinn-motion/tree/main/examples) in the monorepo (vue, svelte, angular, lit, next, astro, expo, …).

## Same control surface

Every painter builds on the core [`Ticker`](/reference/core-api#ticker):

| Method | Does |
| --- | --- |
| `play()` / `pause()` / `stop()` / `toggle()` | transport |
| `seek(t)` | jump to `t` seconds |
| `seekFraction(f)` | jump to `0..1` of the duration |
| `setProgress(p)` | external 0…1 (scroll, gesture, scrubber) |
| `setRate(r)` | playback speed |
| `loop` | get/set looping |

Framework bindings expose the same surface via refs, composables, component methods, or the custom element instance.

## Fidelity at a glance

| Feature | DOM | Canvas | React* | RN |
| --- | :-: | :-: | :-: | :-: |
| Transforms & opacity | ✅ | ✅ | ✅ | ✅ |
| Gradients | ✅ | ✅ | ✅ | ⚠️ |
| SVG vector paths + arrowheads | ✅ | ⚠️ | ✅ | ❌ |
| Masks / clip shapes | ✅ | ⚠️ | ✅ | ⚠️ |
| Procedural shaders | ✅ | ⚠️ | ✅ | ❌ |

\* React / Vue / Svelte / Angular / Lit with `renderer="dom"` inherit DOM fidelity; `renderer="canvas"` inherits Canvas fidelity.

<Note>
  The DOM adapter is the fidelity reference. Canvas and React Native trade some rarer effects for portability.
</Note>
